Install bilang-slidekeys as a bookmarklet

Zero install: drag a button to your bookmarks bar. Click it on any web page to summon a bilingual on-screen keyboard.

Install

Make sure your bookmarks bar is visible (Ctrl+Shift+B in Chrome / Firefox / Edge), then drag this button up to it:

↑ drag this ↑ ⌨ Bilang Keyboard drag to your bookmarks bar
Browsers usually block clicking the button on this page (they treat javascript: bookmarklets specially). That's expected. The bookmarklet works once it's saved as a bookmark — drag it.

Use

  1. Go to any web page (e.g. Wikipedia, your email, a chat app)
  2. Click into a text input or textarea
  3. Click the ⌨ Bilang Keyboard bookmark in your bookmarks bar
  4. The keyboard slides up. Type on your physical keyboard — Cyrillic appears in the input (LiveType is the default mode)
  5. Click the bookmark again to hide / show the keyboard

How it works

The bookmarklet itself is just a tiny loader (~250 characters). When you click it, it fetches the full keyboard script from jsDelivr (a CDN that mirrors GitHub) and injects it into the current page.

The script defines a custom HTML element <bilingual-keyboard>, creates one, and adds it to the page's body. The keyboard's UI is fully isolated using Shadow DOM, so it won't conflict with the host site's styles.

Subsequent clicks toggle the keyboard panel's visibility (no re-fetch).

What gets executed

The bookmarklet code is short enough to read in full:

(function () {
  if (window.__bilangSlidekeys__) {
    var k = document.querySelector('bilingual-keyboard');
    if (k) {
      if (k.hasAttribute('visible')) k.removeAttribute('visible');
      else k.setAttribute('visible', '');
    }
    return;
  }
  var s = document.createElement('script');
  s.src = 'https://cdn.jsdelivr.net/gh/grogzoid/bilang-slidekeys@main/bookmarklet/bilang-slidekeys.bookmarklet.js';
  document.head.appendChild(s);
})();

The full bundle that gets fetched is at bookmarklet/bilang-slidekeys.bookmarklet.js in the repo. It's ~50 KB, a few hundred lines.

Privacy

  • The bookmarklet makes one network request the first time you click it on a page: fetching the keyboard script from jsDelivr's CDN.
  • jsDelivr sees that request (URL + your IP). It does not see what you type.
  • The keyboard itself does not make any network requests, store any data beyond the floating button's position (in localStorage), or send anything anywhere.
  • Source code is open: github.com/grogzoid/bilang-slidekeys

Limitations

  • Sites with strict Content Security Policy may block third-party scripts and the bookmarklet won't load. Most everyday sites (Gmail, WhatsApp Web, Wikipedia, Twitter, GitHub) work fine. Some banks and government sites are locked down.
  • You have to click the bookmarklet on every new page. Unlike the userscript or the upcoming Chrome extension, bookmarklets don't run automatically. For a use-on-every-page experience, install the Tampermonkey userscript instead.
  • WhatsApp Web's chat input may not push up properly when the keyboard appears (see INLINE-PUSHUP.md). Use the keyboard's pop-out (⧉) button to put it in a separate Picture-in-Picture window — that sidesteps the issue.

Uninstall

Right-click the bookmark in your bookmarks bar → Delete. That's it.
