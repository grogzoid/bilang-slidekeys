Install bilang-slidekeys as a bookmarklet

Zero install: drag a button to your bookmarks bar. Click it on any web page to summon a bilingual on-screen keyboard.

Install

Make sure your bookmarks bar is visible (Ctrl+Shift+B in Chrome / Firefox / Edge), then drag this button up to it:

↑ drag this ↑ ⌨ Bilang Keyboard drag to your bookmarks bar
Browsers usually block clicking the button on this page (they treat javascript: bookmarklets specially). That's expected. The bookmarklet works once it's saved as a bookmark — drag it.

Use

  1. Go to any web page (e.g. Wikipedia, your email, a chat app)
  2. Click into a text input or textarea
  3. Click the ⌨ Bilang Keyboard bookmark in your bookmarks bar
  4. The keyboard slides up. Type on your physical keyboard — Cyrillic appears in the input (LiveType is the default mode)
  5. Click the bookmark again to hide / show the keyboard

How a bookmarklet works (in plain terms)

A bookmarklet is just a regular browser bookmark — but instead of saving a web address, it saves JavaScript code that runs on whatever page you're currently looking at when you click the bookmark. The browser sees a URL starting with javascript: and treats it as code to execute, not a page to load.

That's the whole concept. There's no installation, no extension, no permissions prompt. Bookmarklets have been a standard browser feature since the late 1990s.

What this specific bookmarklet does

  1. Looks for an existing <bilingual-keyboard> element on the page. If found, toggles its visibility (so re-clicking the bookmark hides/shows the keyboard).
  2. Otherwise, defines a custom HTML element called <bilingual-keyboard>, creates one, and adds it to the page.
  3. The element's UI lives entirely inside a Shadow DOM — fully isolated from the host page's CSS. The keyboard sits at the bottom of the viewport via a CSS overlay; it doesn't modify any of the page's existing content.

Security & transparency

  • Auditable code. The full code that runs is embedded directly in the bookmark URL. You can right-click the installed bookmark → Edit (or Properties), and read every byte of what it does. Decode the URL with any URL-decoder for a more readable view, or read the source on GitHub: github.com/grogzoid/bilang-slidekeys
  • No network requests. The bookmarklet bundle is self-contained — it doesn't load anything from the internet, doesn't ping a CDN, doesn't phone home. The only data persistence is the floating button's screen position, stored locally in your browser's localStorage.
  • No analytics, no telemetry. What you type stays in your browser. Bilang-slidekeys has no ability to see, log, or transmit anything.
  • Same trust model as any web code. A bookmarklet runs with the same privileges as the page you click it on — it can read and modify the DOM of that page, just like any script the site itself loads. It can not access cookies / data from other sites, your local files, or your browser's settings.
  • Trusted on banks/financial sites? Just don't click it there. The bookmarklet only runs when you explicitly click it. We strongly recommend not running this (or any third-party bookmarklet) on sites where you're logged into sensitive accounts.
  • The GitHub repository is the canonical source. Anything you find there is what's in the bookmarklet. We don't have a separate "production" build — the same code in the repo is what's running in your browser.

Updates

Bookmarklets do not auto-update. The code is frozen in the URL at the moment you dragged the button. If we improve the keyboard or fix a bug, you'll need to re-visit this page and re-drag the button to get the new version. (Right-click the old bookmark → Delete first, or just overwrite by dragging on top.)

This is a feature, not a bug — it means the code can never silently change after you install. For a use-everywhere-and-update-automatically experience, see the Tampermonkey userscript distribution instead.

What's actually in the URL

The javascript: URL is roughly 46,000 characters long — the entire bilingual keyboard, minified and URL-encoded into one giant URL. Most of it is the custom element implementation (~32 KB of JavaScript before encoding). You can paste the URL into a URL decoder if you want to read the unencoded JavaScript, or just look at the same code in the repo:

  • src/bilingual-keyboard.js — the web component (custom element)
  • src/layouts.js — the EN ↔ Ukrainian keymap
  • bookmarklet/build.js — the build script that bundles + minifies these into the bookmarklet URL

Limitations

  • You have to click the bookmarklet on every new page. Bookmarklets don't run automatically — that's the whole point. For a use-on-every-page experience, install the Tampermonkey userscript.
  • Browser URL length limits. The inlined bookmarklet is ~46 K characters. Chrome / Edge / modern Firefox handle this fine. Very old browsers or some restrictive enterprise builds may truncate the URL.
  • WhatsApp Web's chat input may not push up properly when the keyboard appears (see INLINE-PUSHUP.md). Use the keyboard's pop-out (⧉) button to put it in a separate Picture-in-Picture window — that sidesteps the issue.
  • Doesn't survive page reload. Click the bookmark again after a page refresh.

Uninstall

Right-click the bookmark in your bookmarks bar → Delete. That's it.
